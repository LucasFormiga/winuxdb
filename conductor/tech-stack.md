# Tech Stack: WinuxDB\n\n## Core Stack\n- **Framework**: [Next.js](https://nextjs.org/) (v16+) with App Router for server-side rendering and static site generation.\n- **Language**: [TypeScript](https://www.typescriptlang.org/) for type safety.\n- **UI Library**: [React](https://react.dev/) (v19) for building interactive user interfaces.\n\n## Styling & Components\n- **CSS**: [TailwindCSS](https://tailwindcss.com/) (v4+) for utility-first styling.\n- **Component Library**: [Shadcn UI](https://ui.shadcn.com/) (built on [Radix UI](https://www.radix-ui.com/)) for high-quality, accessible UI components.\n- **Icons**: [Lucide-React](https://lucide.dev/) for consistent iconography.\n- **Utilities**: [Class-Variance-Authority](https://cva.style/), [Clsx](https://github.com/lukeed/clsx), and [Tailwind-Merge](https://github.com/dcastil/tailwind-merge) for clean component composition.\n\n## Development & Tooling\n- **Linting & Formatting**: [Biome](https://biomejs.dev/) for fast linting, formatting, and checking.\n- **Testing**: [Vitest](https://vitest.dev/) with [React Testing Library](https://testing-library.com/docs/react-testing-library/intro/) for unit and integration testing.\n- **Package Manager**: [NPM](https://www.npmjs.com/) (via package-lock.json).\n\n## Localization\n- **Multi-language**: Planned support for English (default), Brazilian Portuguese, and Spanish. (Final library/method to be determined during implementation).\n\n## Design System\n- **Mobile-first**: Prioritize mobile device usability and responsiveness.\n- **Customization**: Specialized customization of Shadcn components to achieve a unique brand identity.

# Tech Stack: WinuxDB

## Core Stack
- **Framework**: [Next.js](https://nextjs.org/) (v16+) with App Router for server-side rendering and static site generation.
- **Language**: [TypeScript](https://www.typescriptlang.org/) for type safety.
- **UI Library**: [React](https://react.dev/) (v19) for building interactive user interfaces.

## Styling & Components
- **CSS**: [TailwindCSS](https://tailwindcss.com/) (v4+) for utility-first styling.
- **Component Library**: [Shadcn UI](https://ui.shadcn.com/) (built on [Radix UI](https://www.radix-ui.com/)) for high-quality, accessible UI components.
- **Icons**: [Lucide-React](https://lucide.dev/) for consistent iconography.
- **Utilities**: [Class-Variance-Authority](https://cva.style/), [Clsx](https://github.com/lukeed/clsx), and [Tailwind-Merge](https://github.com/dcastil/tailwind-merge) for clean component composition.

## Development & Tooling
- **Linting & Formatting**: [Biome](https://biomejs.dev/) for fast linting, formatting, and checking.
- **Testing**: [Vitest](https://vitest.dev/) with [React Testing Library](https://testing-library.com/docs/react-testing-library/intro/) for unit and integration testing.
- **Package Manager**: [NPM](https://www.npmjs.com/) (via package-lock.json).

## Localization
- **Multi-language**: [next-intl](https://next-intl-docs.vercel.app/) for internationalization, supporting English (default), Brazilian Portuguese, and Spanish.

## Design System
- **Mobile-first**: Prioritize mobile device usability and responsiveness.
- **Customization**: Specialized customization of Shadcn components to achieve a unique brand identity.
